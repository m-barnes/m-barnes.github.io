Content Categories

Each HTML element must abide by rules defining what kind of content it can have. These rules are grouped into content models common to several elements. Each HTML element belongs to zero, one, or multiple content models, each setting rules that the element's content must follow in an HTML-conformant document.

There are three types of content categories:

  • Main content categories, which describe common content rules shared by many elements
  • Form-related content categories, which describe content rules common to form-related elements
  • Specific content categories, which describe rare categories shared only by a few elements, sometimes only in a specific context

Some elements in content categories overlap, fulfilling multiple conditions and roles. Some elements are only included in some content categories if--and only if--other related elements are present.


Metadata Content

These elements modify the presentation or behavior of the rest of the document, link to other documents or other out of band information.

  • <base> - Specifies a base URL for all additional URLs in the document.
  • <meta> - Used for metadata.
  • <link> - Specifies relationship between the current document and an outside source. Mostly used to link stylesheets.
  • <noscript> - Used to specify code that should run if JavaScript isn't supported or turned off.
  • <script> - Used for executable script or to reference an executable script.
  • <style> - Used for style information, usually CSS.
  • <title> - Used for the title of the document. Only supports text.

Flow Content

These elements are typically text or other embedded content. Here are a few.

  • <a> - Anchor element. Used primarily for linking to other elements or outside resources.
  • <article> - Represents a separate, self-contained composition in a document.
  • <aside> - A section of the page that is tangentially related to the main content, but could be considered separate.
  • <audio> - Used to embed audio element.
  • <blockquote> - Offsets an extended quotation.
  • <br/> - Creates a line break in the text.
  • <button> - Clickable button.
  • <div> - Document division element. This is a generic tag used when nothing else fits.
  • <em> and <i> - Emphasis and italics. Emphasis elements can be nested.
  • <fieldset> - Groups control elements in a form.
  • <figure> - Separate element usually paired with a caption (e.g. illustrations, images, code snippets, etc.)
  • <footer> - The footer section of the page.
  • <form> - Defines a form.
  • <h1> - <h6> - Heading elements of various sizes. <H1> being the largest.
  • <header> - Represents a group of introductory elements or navigational aids.
  • <hr/> - Horizontal rule. Throws a line right across the page to represent a thematic break in content.
  • <img> - Images.
  • <label> - Captions for interactive user elements like in a form.
  • <main> - Used to separate main content from the rest of the page.
  • <nav> - Used to define navigational controls on the page.
  • <ol> and <ul> - ordered and unordered lists. Ordered lists have numbers; unordered lists do not. These are often paired with <li> elements.
  • <p> - Paragraph element.
  • <pre> - Used for preformatted text. Preformatted text is typically displayed exactly as it's laid out in a file. Whitespace is respected.
  • <section> - Represents a generic section of document.
  • <span> - A generic inline container for separating and emphasizing content.
  • <strong> and <b> - Strong and bold.
  • <sub> and <sup>- Subscript and superscript.
  • <table> - Used to wrangle tabular data.
  • <textarea> - Represents a multi-line, plain-text form control.
  • <video> - Offsets video content.

Sectioning Content

These elements create sections of content in the document. There is a lot of overlap with the sectioning content category as you can see.

  • <h1> - <h6> - Heading elements of various sizes. <H1> being the largest.

Heading Content

Heading content defines the title of a section whether marked by an explicit sectioning content element or implicitly defined by the heading content itself.

  • <article> - Represents a separate, self-contained composition in a document.
  • <aside> - A section of the page that is tangentially related to the main content, but could be considered separate.
  • <footer> - The footer section of the page.
  • <header> - Represents a group of introductory elements or navigational aids.
  • <nav> - Used to define navigational controls on the page.
  • <section> - Represents a generic section of document.

Phrasing Content

These elements define text and markup. Runs of phrasing content make up paragraphs.

  • <a> - Only if it contains phrasing content.
  • <audio> - Used to embed audio element.
  • <br/> - Creates a line break in the text.
  • <button> - Clickable button.
  • <em> and <i> - Emphasis and italics. Emphasis elements can be nested.
  • <img> - Images.
  • <label> - Captions for interactive user elements like in a form.
  • <script> - References an executable script.
  • <span> - A generic inline container for separating and emphasizing content.
  • <strong> and <b> - Strong and bold.
  • <sub> and <sup>- Subscript and superscript.
  • <video> - Offsets video content.

Embedded Content

These elements are for controlling and embedding outside resources or inserts content from another markup language.

  • <audio> - Used to embed audio element.
  • <canvas> - Element offsets an area for drawing graphics, usually with JavaScript.
  • <embed> - Integration point for an external application or interactive content.
  • <iframe> - Nested browsing content like a separate html page.
  • <img> - Images.
  • <video> - Offsets video content.

Interactive Content

These elements are for interactive user content.

  • <a> - Anchor element. Used primarily for linking to other elements or outside resources.
  • <audio> - Used to embed audio element. Only if the controls element is present.
  • <button> - Clickable button.
  • <embed> - Integration point for an external application or interactive content.
  • <iframe> - Nested browsing content like a separate html page.
  • <img> - Images. Only if the usemap element is present.
  • <label> - Captions for interactive user elements like in a form.
  • <video> - Offsets video content. Only if the controls element is present.

Palpable Content

Content is palpable when it's neither empty nor hidden. Elements in flow content or phrasing content should have at least one palpable node.

Form Associated Content

Form content exposed by a form attribute (e.g. either a <form> element or an element with a form attribute.

  • <button> - Clickable button.
  • <fieldset> - Groups control elements in a form.
  • <input> - Used to define fields for accepting data from user.
  • <label> - Captions for interactive user elements like in a form.
  • <select> - Represents a selection of options.
  • <textarea> - Represents a multi-line, plain-text form control.
